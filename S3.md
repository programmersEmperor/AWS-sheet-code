# S3 Object Lock and Deletion Protection — Scenario Phrases

Use this as a quick SAA-C03 review table. Every row maps a scenario or requirement to the solution you should think of first.

## Scenario Table

| Scenario / requirement | Solution |
|---|---|
| Recover from accidental object deletion | Enable S3 Versioning; restore by removing the delete marker or using a previous version |
| Recover from accidental object overwrite | Enable S3 Versioning; restore the previous object version |
| Protect objects from both accidental deletion and overwrite | Enable S3 Versioning and MFA Delete |
| Require MFA before permanently deleting object versions | Enable MFA Delete |
| Require MFA before changing bucket versioning state | Enable MFA Delete |
| Stop a specific IAM user or role from deleting objects | Use an IAM policy or S3 bucket policy with an explicit Deny for delete actions |
| Stop accounts in an AWS Organization from deleting S3 objects | Use an SCP with an explicit Deny for delete actions |
| Need WORM storage | Use S3 Object Lock |
| Need immutable object-level protection in S3 | Use S3 Object Lock |
| Prevent an object version from being deleted or overwritten for a fixed retention period | Use S3 Object Lock retention period |
| No user, including the root user, can delete or overwrite an object for a fixed period | Use S3 Object Lock in Compliance mode |
| Privileged users may bypass retention when explicitly authorized | Use S3 Object Lock in Governance mode |
| Emergency break-glass administrators need to override retention | Use Governance mode with bypass permission |
| Strict regulatory retention must not be bypassed by any user | Use Compliance mode |
| Preserve an object because of a legal case or investigation with no known end date | Use S3 Object Lock Legal Hold |
| Preserve an object until the legal team manually releases it | Use Legal Hold |
| Prevent logs or tax records from deletion or overwrite for compliance | Use S3 Object Lock; choose Compliance mode for strict fixed retention or Legal Hold when no end date is given |
| Restrict S3 data access requests to a specific VPC only | Configure an S3 Access Point restricted to that VPC |
| Restrict access to a specific VPC and prevent deletion or overwrite with WORM protection | Configure an S3 Access Point restricted to the VPC and enable S3 Object Lock; use Legal Hold if no fixed retention period is stated |
| EC2 instances in a private subnet must access S3 without using the public internet or NAT | Use an S3 VPC endpoint |
| Private subnet resources need private access to both S3 and DynamoDB | Use S3 and DynamoDB VPC endpoints |
| Restrict an S3 bucket so requests must come through a specific VPC endpoint | Use a bucket policy with the `aws:SourceVpce` condition |
| Restrict an S3 bucket so requests must come from a specific VPC | Use a bucket policy with the `aws:SourceVpc` condition |
| Give different applications or teams different access paths to the same bucket | Use S3 Access Points |
| Need member-only access to multiple private files through CloudFront without changing URLs | Use CloudFront signed cookies |
| Need member-only access to one private file or URL | Use CloudFront signed URL |
| Browser blocks JavaScript requests from S3 website endpoint to S3 API endpoint | Configure S3 CORS |
| Keep old objects but move them to cheaper storage after a period | Use S3 Lifecycle transition |
| Automatically delete current objects after a period | Use S3 Lifecycle expiration |
| Automatically delete old noncurrent versions after a period | Use S3 Lifecycle NoncurrentVersionExpiration |
| Need to archive documents from on-premises file shares to S3 and Glacier with file interface | Use AWS Storage Gateway File Gateway plus S3 Lifecycle |
| Need to detect PII or privacy risks in S3 buckets | Use Amazon Macie |
| Need ad-hoc SQL queries on S3 data with least operational overhead | Use AWS Glue Crawler and Amazon Athena |
| Need to transform JSON/CSV data in S3 into Parquet for analytics performance | Use AWS Glue ETL or equivalent transformation to Parquet |
| Need to process only new or changed S3 data in repeated Glue jobs | Enable AWS Glue job bookmarks |
| Need private S3 access to VPC | S3 Access Point or bucket policy conditions |
| Need WORM but the answer disables Versioning | Wrong; S3 Object Lock requires Versioning |
| Need fixed retention but the answer says Legal Hold for one year | Wrong; fixed duration means Object Lock retention period |
| Need unknown legal retention but the answer says one-year retention | Wrong; unknown legal duration means Legal Hold |
| Need deletion protection but the answer says Lifecycle | Wrong; Lifecycle deletes or transitions objects for cost management |
| Need recovery from accidental delete but no strict WORM requirement | Use Versioning, not Object Lock |
| Need prevention of deletion or overwrite during retention | Use Object Lock, not Versioning alone |

## One-Line Rules

| Scenario / requirement | Solution |
|---|---|
| Recovery after delete or overwrite | Versioning |
| Harder permanent deletion of versions | MFA Delete |
| WORM or immutable storage | Object Lock |
| Privileged users can bypass | Governance mode |
| Nobody can bypass, including root | Compliance mode |
| No fixed end date because of legal/investigation need | Legal Hold |
| Fixed retention duration | Object Lock retention period |
| VPC-only access path to S3 data | S3 Access Point restricted to a VPC |
| Private subnet to S3 without internet or NAT | S3 VPC endpoint |
| Restrict bucket access to a specific VPC endpoint | Bucket policy with `aws:SourceVpce` |
| Restrict bucket access to a specific VPC | Bucket policy with `aws:SourceVpc` |
